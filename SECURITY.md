# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.2.x   | Yes       |
| < 0.1.0 | No        |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue.

Report it directly to: dex-hon-sci@outlook.com

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

You can expect an acknowledgement within 72 hours.

## Scope

The following are considered security issues in this project:

- Credential or token exposure
- Order parameter bypass or injection
- Pre-trade risk check bypass
- Unauthorised access to account or position data

## Out of Scope

- Vulnerabilities in CQG's own infrastructure or WebAPI
- Issues in third-party dependencies (report those upstream)
- Anything requiring physical access to the machine
